## v1.0.0 — Self-hosted RTMP restreamer, first stable release

One stream from OBS in, fanned out to Twitch, YouTube Live, Kick, and
VK Video Live. Web control panel with a single START/STOP and
per-target settings. Ships as one multi-arch container (linux/amd64 +
linux/arm64), pulled from GitHub Container Registry, cosign-signed.

### Highlights

- **Multi-platform fanout** — one OBS push, N enabled targets, each
  `ffmpeg -c copy` (no transcoding, no quality loss).
- **Web control panel** — React SPA with live status over WebSocket,
  per-target enable/disable, ingest-key rotation, audit log.
- **Encrypted secret storage** — stream keys at rest under AES-256-GCM
  with a key derived from `RESTREAM_MASTER_PASSPHRASE` (Argon2id KDF).
- **First-boot admin bootstrap** — `RESTREAM_ADMIN_PASSWORD` is
  **optional**. When unset on first boot, the container autogenerates
  a 20-char admin password and prints it to stdout exactly once. See
  [ADR-0005 §"Bootstrap"](../architecture/ADR-0005-auth-model.md).
- **VK Video Live as a first-class target** — including VK's
  single-use-key constraint; the panel prompts for the key at START
  time and clears it after the session.
- **Supply-chain integrity** — every image cosign-keyless-signed via
  GitHub Actions OIDC; `grype` HIGH+-with-fix blocks the publish.
- **Schema versioning** — `SCHEMA_VERSION = 2`. The container refuses
  to start against a higher on-disk version than the binary knows
  (no silent downgrades).

### Installation

```bash
docker run -d \
  --name restream-plus \
  -p 1935:1935 -p 8000:8000 \
  -v restream-data:/data \
  -e RESTREAM_MASTER_PASSPHRASE='change-me-to-something-long-and-random' \
  ghcr.io/pelmentor/restream-plus:v1.0.0
```

Full operator guide: [docs/ops/deployment.md](../ops/deployment.md).

### Pin recommendation

Pin `:v1.0.0` (immutable). `:latest` and `:v1` are convenience aliases
and will float to subsequent patch releases.

### Verify the image signature

```bash
cosign verify \
  --certificate-identity-regexp '^https://github\.com/pelmentor/Restream_Plus/\.github/workflows/release\.yml@refs/tags/v[0-9.]+$' \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  ghcr.io/pelmentor/restream-plus:v1.0.0
```

Requires `cosign` **2.2+** (matches the floor stated in
[deployment.md](../ops/deployment.md) and
[release-checklist.md](../ops/release-checklist.md) step 11). The
signing job in CI uses `sigstore/cosign-installer@v4.1.2`, which
provisions a current cosign 2.x release.

### Breaking changes vs. `:edge`

None for fresh installs.

For operators who already have a `/data` volume from an `:edge` pull:
`SCHEMA_VERSION` is now `2` (was `1`). Per project Rule №2 (no
migration baggage), there is no in-place migration. Use the panel's
**Settings → Export** flow to bundle your configuration off the old
container, start `:v1.0.0` against a fresh volume, then **Settings →
Import** the bundle. Full step-by-step:
[docs/ops/upgrade.md §"Schema-bump releases"](../ops/upgrade.md).

### Known characteristics worth flagging

- **First-boot password is shown ONCE.** Capture it from the container
  log (`docker logs restream-plus 2>&1 | grep -A 8 'FIRST BOOT'`)
  before restarting, or set `RESTREAM_ADMIN_PASSWORD` to skip autogen
  entirely. Lost-password recovery uses
  `RESTREAM_RESET_ADMIN_PASSWORD=1` paired with a new
  `RESTREAM_ADMIN_PASSWORD` on the next start.
- **VK single-use keys are by VK's design, not ours.** The Dashboard
  blocks START on an enabled VK target with no key and prompts inline.
  This is documented in
  [platforms-reference.md §"VK Video Live"](../architecture/platforms-reference.md).
- **`:edge` keeps moving.** It is published on every push to `main`
  and is not signed for end users; use `:vX.Y.Z` in production.
