## v1.1.1 — Hotfix: serve the React SPA from the FastAPI app

Critical hotfix. Both v1.0.0 and v1.1.0 shipped without a static-file
mount for the bundled React SPA — `GET /` returned the JSON
`{"detail":"Not Found"}` instead of `index.html`, making the web
control panel **unreachable in a browser**. The JSON API itself was
working (auth, REST, WS, internal webhooks), so this affected only
the operator UI surface. The bug surfaced during the first real-world
post-install browser test of `:v1.1.0` and is fixed here.

### What changed

- **`app/main.py` mounts the SPA** at the catch-all path after every
  API router. On `GET /` and any path that doesn't collide with the
  API surface, the handler returns either the matching on-disk file
  from `/app/web/dist/` (so `/assets/*.js`, `/favicon.ico` etc.
  serve the real bytes) or `index.html` as a history-API fallback
  (so deep links like `/settings/security` survive a hard refresh).
- **Defensive `/api/`, `/internal/`, `/ws`, `/livez`, `/readyz`
  exclusion** inside the catch-all so a 404 on those paths stays a
  404 — silently returning the SPA shell would hide frontend fetch
  typos behind an HTML body.
- **Path-traversal defense** — requests that resolve outside the
  SPA dir (`..`-escaped URLs) get 404, not the targeted file.
- **8 new tests** at `tests/api/test_spa_serving.py` pin the
  contract so a future refactor can't silently drop the mount
  again.

### Upgrade

```bash
docker pull ghcr.io/pelmentor/restream-plus:v1.1.1
docker stop -t 30 restream-plus
docker rm restream-plus
docker run -d \
  --name restream-plus \
  --restart unless-stopped \
  -p 1935:1935 -p 8000:8000 \
  -v /mnt/user/appdata/restream-plus:/data \
  --stop-timeout 30 \
  -e RESTREAM_MASTER_PASSPHRASE='same-passphrase-as-before' \
  ghcr.io/pelmentor/restream-plus:v1.1.1
```

Volume + DB + encrypted credentials persist across the recreate.

### Postmortem note

This bug should have been caught by release-checklist step 12
(real-RTMP smoke — human-required, "humans verify the data path"),
but step 12 was never executed for v1.0.0 or v1.1.0. Step 12 now
explicitly includes browser reachability of the panel.
