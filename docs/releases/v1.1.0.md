## v1.1.0 — Live dashboard stats

Live operator stats during a session: per-target outbound bitrate +
dropped frames, host process-tree CPU%, and ingest bitrate from
nginx-rtmp. Always-visible header chips for at-a-glance health, hero
ingest/egress pair + 60s sparkline for the showpiece, per-target tile
metrics for "which target is broken." Builds on v1.0.0 with NO
breaking changes and NO schema bump.

### Highlights

- **Per-target bitrate + drops** — replace the placeholder
  `WORKER / FAILURES` row in each target tile with real Mbps + ffmpeg
  drop_frames pulled from the existing progress stream. The `Sparkline`
  in the slide-out details now plots a 60s rolling buffer of that
  target's egress.
- **Hero ingest/egress pair + sparkline (LIVE state)** — operator
  diagnostic: comparing what's coming IN from OBS vs what's going OUT
  to platforms answers "is it me or is it Twitch" at a glance.
  Aggregate sparkline reuses the same canvas component.
- **Always-visible header chips** — `Egress N.N Mbps` + `CPU N%` next
  to the run-state badge. Hidden when offline (no lie). CPU debounced
  color bands: default <70%, warn 70-90%, error >90%.
- **Process-tree CPU%**, NOT system-wide — sum of FastAPI control
  plane + each live ffmpeg pid, normalized to 0-100% of one host
  (divided by `os.cpu_count()`). Operator wants "is Restream_Plus
  eating my host," not "is my box busy."
- **`/proc/<pid>/stat` direct read** — no `psutil` dependency. CPU
  accounting baseline is recomputed each tick with a defense against
  pid-reuse upward-bogons (cap at `100 * logical_cpus`).
- **Defense-in-depth on the XML parser** — `defusedxml.ElementTree`
  for nginx-rtmp `rtmp_stat`. The source is loopback today but the
  XXE / billion-laughs attack class is eliminated outright.
- **Version-string drift fixed** — `pyproject.toml`, `app/version.py`,
  `app/main.py`, `web/package.json` now all report `1.1.0` matching
  the operator-facing tag. v1.0.0 shipped with the in-code version
  still reading `0.1.0`; `/api/about` and `app_started` audit rows
  are now consistent with the image tag.

### Installation

```bash
docker run -d \
  --name restream-plus \
  -p 1935:1935 -p 8000:8000 \
  -v restream-data:/data \
  -e RESTREAM_MASTER_PASSPHRASE='change-me-to-something-long-and-random' \
  ghcr.io/pelmentor/restream-plus:v1.1.0
```

Full operator guide: [docs/ops/deployment.md](../ops/deployment.md).

### Pin recommendation

Pin `:v1.1.0` (immutable). `:latest` and `:v1` are convenience aliases
that float to subsequent releases.

### Verify the image signature

```bash
cosign verify \
  --certificate-identity-regexp '^https://github\.com/pelmentor/Restream_Plus/\.github/workflows/release\.yml@refs/tags/v[0-9.]+$' \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  ghcr.io/pelmentor/restream-plus:v1.1.0
```

Requires `cosign` **2.2+**. The signing job in CI uses
`sigstore/cosign-installer@v4.1.2`, which provisions a current
cosign 2.x release.

### Upgrade path

In-place upgrade from `:v1.0.0` is supported with no schema bump:

```bash
docker pull ghcr.io/pelmentor/restream-plus:v1.1.0
docker stop -t 30 restream-plus
docker rm restream-plus
docker run -d \
  --name restream-plus \
  -p 1935:1935 -p 8000:8000 \
  -v restream-data:/data \
  -e RESTREAM_MASTER_PASSPHRASE='same-passphrase-as-before' \
  ghcr.io/pelmentor/restream-plus:v1.1.0
```

Full procedure: [docs/ops/upgrade.md](../ops/upgrade.md).

### Breaking changes vs. v1.0.0

None. `SCHEMA_VERSION` is unchanged at `2`. WS protocol version
unchanged at `1` (the new `host.stats` event and the new
`target.snapshot.last_progress` field are additive — old clients
silently ignore them).

### What changed under the hood

- New backend module `app/fanout/host_stats.py` — `HostStatsSampler`
  + `ProcfsStatReader` + `NginxRtmpStatFetcher`. 2s sampling tick.
- New `HostStatsEvent` on the supervisor's bounded event bus.
- `WorkerSnapshot` gets optional `last_progress` field carrying the
  last ffmpeg progress frame (bitrate, fps, drop_frames, speed).
- Nginx config exposes loopback `/rtmp_stat` for ingest bitrate.
- New runtime dependency `defusedxml`.
- ADR-0011 §"Open questions" expanded with the WS-vs-`/metrics`
  trade-off (3 reasons: consumer count, auth posture, source-of-
  truth duplication). ADR-0003 §"Open questions" notes
  single-drainer-vs-`/metrics` as the real blocker for that future
  amendment.

### Known characteristics worth flagging

- **Live-stats UI does NOT render when no session is running.** The
  header chips hide on `offline`; the hero sparkline appears only in
  `live`. This is intentional — surfacing "0 Mbps" when there's
  nothing to measure would lie about a dark show.
- **Ingest bitrate reads as "—" when nginx-rtmp is unreachable or no
  publisher is connected.** Same reason — a 0 would lie.
- **CPU is process-tree, NOT system-wide.** A noisy neighbor on the
  same host won't show up in the dashboard chip. By design — the
  operator wants Restream_Plus's footprint, not the box's.
- **No `/metrics` endpoint.** The dashboard is the only consumer.
  See ADR-0011 §"Open questions" for the trade-off and when to add
  one (operator wires Grafana = then, not before).
- **First-boot password is shown ONCE on stdout** (unchanged from
  v1.0.0). Capture it with `docker logs restream-plus 2>&1 | grep -A 8
  'FIRST BOOT'` or set `RESTREAM_ADMIN_PASSWORD` to skip autogen.
